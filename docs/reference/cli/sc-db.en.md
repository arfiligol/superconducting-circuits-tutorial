# sc-db

Single entrypoint for database management, using model subcommands.

## Model CRUD Scope

| Model | List | Get | Create | Update | Delete | Notes |
|---|---|---|---|---|---|---|
| DatasetRecord | ✅ | ✅ | ❌ | ❌ | ✅ | Created by import pipeline |
| DataRecord | ✅ | ✅ | ❌ | ❌ | ✅ | Created by import pipeline |
| DerivedParameter | ✅ | ✅ | ❌ | ❌ | ✅ | Created by analysis pipeline |
| Tag | ✅ | ✅ | ✅ | ✅ | ✅ | Manual maintenance and rename allowed |

## Usage

```bash
uv run sc-db <model> <action> [args]
```

## Arguments

| Argument | Description | Default |
|---|---|---|
| `<model>` | `dataset-record` / `data-record` / `derived-parameter` / `tag` | - |
| `<action>` | Depends on model (see subpages) | - |

## Options

| Option | Description | Default |
|---|---|---|
| `--help` | Show help | - |

## Examples

**List DatasetRecord**
```bash
uv run sc-db dataset-record list
```

**Show DatasetRecord**
```bash
uv run sc-db dataset-record info <ID_OR_NAME>
```

**Delete DataRecord**
```bash
uv run sc-db data-record delete <ID>
```

**Create Tag**
```bash
uv run sc-db tag create "NewTag"
```

**Rename Tag**
```bash
uv run sc-db tag update "NewTag" "RenamedTag"
```

## Notes

!!! note "Deprecated"
    `sc-db list/info/delete` are removed. Use `sc-db <model> <action>`.
