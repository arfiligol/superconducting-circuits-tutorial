# sc analysis fit

Run SQUID LC fitting (formerly `sc-fit-squid`).

## Usage

```bash
uv run sc analysis fit lc-squid [OPTIONS] [datasets ...]
```

## Arguments

| Argument | Description | Default |
|---|---|---|
| `components` | Dataset names or IDs | Built-in defaults |

## Options

| Option | Description |
|---|---|
| `--modes` | Modes to fit/plot (e.g. `Mode 1`) |
| `--title` | Plot title |
| `--ls-min` / `--ls-max` | Ls bounds (nH) |
| `--c-min` / `--c-max` | C bounds (pF) |
| `--fixed-c` | Fixed capacitance value (pF) |
| `--fit-min` / `--fit-max` | Fit window (GHz) |
| `--save-to-db` | Persist fit outputs to SQLite (`DataRecord` + `DerivedParameter`) |
| `--device-type` | Device type for persisted `DerivedParameter` rows |
| `--replace-existing` / `--append` | Replace or append existing persisted fit outputs |

## Examples

**Basic Fit (Default: With Ls)**
```bash
uv run sc analysis fit lc-squid LJPAL658_v1
```

**No Ls Fit**
```bash
uv run sc analysis fit lc-squid --no-ls LJPAL658_v1
```

**With Ls and upper bound**
```bash
uv run sc analysis fit lc-squid --ls-max 0.2 LJPAL658_v1
```

**Fixed C fit (C=1.45 pF)**
```bash
uv run sc analysis fit lc-squid --fixed-c 1.45 LJPAL658_v1
```

**Filter Specific Modes (Repeat flag)**
```bash
uv run sc analysis fit lc-squid --modes 1 --modes 2 LJPAL658_v1
```

**Fit and persist to DB**
```bash
uv run sc analysis fit lc-squid LJPAL658_v1 --save-to-db --device-type qubit
```
